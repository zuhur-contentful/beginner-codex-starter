What you're going to build

By the end of this guide you will have built a real web app — something that runs in a browser, saves data, and uses AI. Something you want to use, not a tutorial project.

Examples of things beginners have built:

  • A private journal that summarises your week using GPT
  • A habit tracker that sends you an encouraging message each morning
  • A reading list with AI-generated summaries of each book
  • A recipe organiser that suggests meals based on what's in the fridge
  • A tool that emails a daily digest of things you care about
The new rule with AI You can now ask AI to set up your machine, plan your project, write the first version of every feature, explain every error, and review your code. Your job is to understand what it's doing — not to memorise syntax.

Your tools

OpenAI makes several products. They're different and the naming is confusing. Here's what you need:

ChatGPT desktop app
Chat interface. Lives in your dock. Cannot edit your files.
→ Brainstorming, questions, planning, learning
Codex App ★ Start here
GUI coding agent. Reads your files, writes code, runs commands, has Git built in.
→ Actually building things
Codex CLI
Same as the Codex App but in your terminal.
→ Same as above, keyboard-first
VS Code + ChatGPT extension
Chat panel inside your editor.
→ Inline help without switching windows
ChatGPT is not deterministic The same question gives a different answer each time. Great for learning. Never paste code you don't understand — you won't be able to fix it when it breaks.

You need a ChatGPT Plus subscription ($20/month) for all of these. The Codex App uses OAuth login — no API key needed to build with it.

Day 1 checklist — get set up in 15 minutes

How to open this guide on any computer Find the file called guide.html in your project folder and double-click it. Opens in your browser. To save as PDF: File → Print → Save as PDF.
  1. Go to github.com/zuhur-contentful/beginner-codex-starter
  2. Click the green Use this template button → Create a new repository → give it a name → click Create repository
  3. Download it to your computer:
    • Click the green Code button → Download ZIP → unzip it
    • Or if you have Git: git clone https://github.com/YOUR-USERNAME/YOUR-REPO.git
  4. Download and open the Codex App from github.com/openai/codex/releases → click Open folder → select the folder you just downloaded
  5. Edit CLAUDE.md — replace PROJECT_DESCRIPTION with one sentence about what you want to build
  6. In Terminal: cp .env.example .env — open .env and fill in SECRET_KEY using the command shown
  7. Create a virtual environment and install packages:
    python3 -m venv venv && source venv/bin/activate && pip install -r requirements.txt
  8. Run tests to confirm everything works: python -m pytest tests/ -v — should see 1 passed
  9. In the Codex App, type: "I want to build [describe your idea]. Help me plan the first feature."
Stuck on any step? Open the ChatGPT desktop app and say: "I'm a complete beginner on Mac. I'm on step [N] of a setup checklist and it isn't working: [paste error]."

When things break

  1. Read the full error message — the last line is usually the actual problem
  2. Google the exact error message in quotes
  3. Say "debug this" in the Codex App — paste the full error and describe what you were doing
  4. Make one change at a time
  5. Run again and see what changed
  6. Repeat

Quick reference

# Activate virtual environment (every time you open a new terminal)
source venv/bin/activate

# Install packages
pip install <package-name>
pip freeze > requirements.txt    # save so others can install too

# Run your app
python main.py

# Run tests
python -m pytest tests/ -v

# Save your work
git add <specific files>         # never: git add .
git commit -m "feat: what you did"
git push
